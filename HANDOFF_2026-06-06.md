## Technical Analysis of the Blank Screen Crash

The crash occurs during the mounting lifecycle of `<AppProvider>` or standard initialization hooks. Because the crash happens in the rendering or effect queue, standard error reporting may be suppressed.

### Potential Root Causes:
1. **IndexedDB / Storage Initialization Failures**:
   - Inside [appLifecycle.ts](file:///Users/kai21/Projects/Other/coaster_credits/context/appLifecycle.ts#L168) and [storage.ts](file:///Users/kai21/Projects/Other/coaster_credits/services/storage.ts), `initializeAndSyncApp` calls `storage.migrateFromLocalStorage()` and several `storage.get` calls.
   - If `indexedDB.open` fails or rejects a promise without catch handlers, this can bubble up and break rendering completely.
2. **Importmap / React Hook Collisions**:
   - The [index.html](file:///Users/kai21/Projects/Other/coaster_credits/index.html#L64) contains a client-side `<script type="importmap">` referencing React/ReactDOM `v19.2.0` from `aistudiocdn.com`.
   - Since Vite compiles code into standard ES bundles and packages dependencies internally (unless explicitly marked as external), this could cause a collision where React is instantiated twice (once bundled in `index-CENOPmHz.js` and once resolved by importmaps), which throws a React runtime exception immediately.
3. **Missing `/index.css` Warning**:
   - The browser console outputs a `404 Not Found` for `/index.css` (referenced at [index.html:L78](file:///Users/kai21/Projects/Other/coaster_credits/index.html#L78)). While non-blocking (Tailwind runs via CDN), this should be removed or cleaned up.
